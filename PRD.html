家庭账本应用 PRD

产品需求文档 · 版本 v1.1

1. 项目概述

1.1 项目背景

为管理家庭收支、记录工资分配而设计的单页Web应用，采用Apple风格UI设计。

1.2 用户角色

家庭成员，工资在每月16号发放

2. 功能需求

2.1 核心概念

  • 发薪日：每月16号（可设置）
  • 日常预算：固定每月6300元
  • 收入分配逻辑：工资到账后，分配为三部分 - 日常预算、上月额外支出、交给老婆

2.2.1 首页

核心功能

  • 当前周期卡片：显示周期日期、本期收入、日常预算、上月额外支出、交给老婆
  • 快速操作区：记收入、记支出（两个按钮并排）
  • 支出列表区：最近支出记录，只显示最近两个月，更早的可折叠展开
  • 每条支出记录显示：图标、分类、日期、金额、编辑/删除按钮

周期显示逻辑：

  • 今日5月12日（<16号）：当前周期显示 4月16日～5月15日（正在进行的周期）
  • 今日5月16日及以后：当前周期显示 5月16日～6月15日

金额计算逻辑：

  • 本期收入：记录在当前周期内的收入
  • 日常预算：固定6300元
  • 上月额外支出：上一个完整周期的额外支出总和
  • 交给老婆：本期收入 - 6300 - 上月额外支出

2.2.2 记收入

流程：

  1. 点击「记收入」
  2. 输入金额、来源、日期
  3. 预览上月额外支出、分配结果（日常6300 + 上月额外 + 老婆 = 总收入）
  4. 保存

2.2.3 记支出

流程：

  1. 点击「记支出」
  2. 输入金额、选择支出类型（日常/额外）
  3. 选择分类、日期、备注（可选）
  4. 保存

2.2.4 统计页面

  • 年份选择器：可切换年份
  • 年度汇总卡片：总收入、日常支出、额外支出、交给老婆
  • 月度趋势区：按倒序只显示有数据的月份，每个月的三色堆叠图（黄色日常、粉色额外、紫色老婆）+ 金额标签

2.2.5 收入列表页面

  • 按年份分组显示所有历史收入记录
  • 每条记录：图标、来源、日期、金额、编辑/删除按钮

2.2.6 额外支出分类页面

  • 年份选择器
  • 显示选中年份的额外支出分类统计，带进度条

2.2.7 设置

  • 设置日常预算（默认6300）
  • 设置发薪日（默认16号）
  • 清除所有数据（危险操作）

3. UI设计规范

3.1 设计风格

  • Apple风格：简洁、圆角、柔和阴影
  • 主题色：蓝色主色、绿色成功、红色错误/删除、紫色老婆、黄色日常、粉色额外

3.2 页面布局

  • 响应式单栏：最大宽度600px，居中显示
  • 卡片式设计：所有内容区域用白色卡片包裹，圆角16px
  • 导航：顶部4个标签切换页面

3.3 交互设计

  • 按钮：小圆角，正常大小按钮，编辑蓝色、删除红色
  • 列表项：左图标、中间信息、右金额+按钮横向排列
  • 折叠展开：三角形图标，点击切换

4. 数据存储

4.1 本地存储

  • 使用浏览器localStorage
  • Key：familyBudget_v2
  • 刷新页面数据不丢失

4.2 数据结构

{
  settings: { dailyBudget: 6300, payday: 16 },
  incomes: [ { id, amount, source, date, allocatedDaily, allocatedExtra, allocatedWife } ],
  expenses: [ { id, amount, type, categoryId, categoryName, categoryEmoji, date, note } ],
  categories: {
    daily: [ { id, name, emoji } ],
    extra: [ { id, name, emoji } ]
  }
}

5. 使用流程

5.1 发薪后的记账流程

  1. 5月16日收到工资 → 点击「记收入」
  2. 输入工资金额，系统自动显示上一个周期（4月16日～5月15日）的额外支出
  3. 计算老婆部分：收入 - 6300 - 上月额外支出
  4. 日常支出随时记，走「记支出」-「日常」
  5. 额外支出随时记，走「记支出」-「额外」

5.2 查看流程

  • 首页看当前周期和最近支出
  • 统计页看年度汇总和趋势
  • 收入列表页看历史收入
  • 额外分类页看额外支出构成
