## 核心逻辑（未改业务公式）

### 周期计算

- 发薪日可设置（默认 16 号）
- 今天 < 发薪日：当前周期为「上月发薪日～本月发薪日前一天」
- 今天 ≥ 发薪日：当前周期为「本月发薪日～下月发薪日前一天」

### 金额计算

```
交给老婆 = max(0, 本期收入 - 日常预算 - 上月额外支出)
```

- 日常预算、发薪日来自设置（不再写死 6300 / 16，统计按设置中的日常预算计月固定额）

## 日常更新要怎么发布？（Netlify / Supabase）

### Netlify — 改了网页就要重新发布

每次修改 `index.html`、`style.css`、`app.js`、`boot-cloud.js` 等前端文件后：

1. 打开 Netlify 项目页（savetofree）
2. 在 **「Drag and drop your project folder」** 再拖一次**项目文件夹本身**（文件夹里**直接**能看到 `index.html`，不要拖它的上一级目录，否则首页会 404）
   - 正确：拖 `save to free project`（打开后第一层就有 `index.html`、`app.js`、`vendor/`）
   - 错误：拖外层父文件夹（发布后地址会变成 `/save%20to%20free%20project/`，根路径 404）
   - 须包含 `vendor/supabase.min.js` 与已填写的 `supabase-config.js`
3. 等显示 Published
4. 手机 / 电脑打开 [https://savetofree.netlify.app](https://savetofree.netlify.app) ，建议强刷或清缓存

**不需要**因为改文案、改样式、改记账逻辑而去改 Supabase。

### Supabase — 多数时候不用动


| 情况                          | 要不要改 Supabase                                                       |
| --------------------------- | ------------------------------------------------------------------- |
| 改界面、改计算、改文案                 | **不用**                                                              |
| 只更新 Netlify 上的前端            | **不用**                                                              |
| 换了 Netlify 域名 / 自定义域名       | **要**：Authentication → URL Configuration 改 Site URL 和 Redirect URLs |
| 第一次建库 / 加新表、改表结构            | **要**：在 SQL Editor 执行新的 `schema.sql` 片段                             |
| 换了 Supabase 项目（新 URL / Key） | **要**：改 `supabase-config.js` 并重新部署 Netlify                          |
| 调整登录方式（如关闭邮箱确认）             | **要**：Authentication → Providers / URL Configuration                |


### 推荐流程（以后加功能）

1. 本地 `node server.js`，用 [http://localhost:8080](http://localhost:8080) 试好
2. 满意后整文件夹拖到 Netlify 发布
3. 用手机打开线上地址验证登录与同步
4. 大改前可在 `_archive/` 再存一份快照（与 v1.0 时一样）
